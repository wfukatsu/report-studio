## 発見した差異・課題

### DIFF-01（P1）: 非居住者チェックボックス「38万円以上の支払」が欠落

**箇所**: Section B（控除対象扶養親族）行 1〜4 の「非居住者である親族」列  
**PDF の実際の内容**: 4 チェックボックス
1. 16歳以上30歳未満又は70歳以上
2. 留学
3. 障害者
4. **38万円以上の支払** ← 現テンプレートに存在しない

**現在のコード**（`buildPersonRow` 内、非居住者エリア）:
```typescript
checkbox(nbX, rowY + 1, 3.0)   // 16歳以上30歳未満または70歳以上
checkbox(nbX, rowY + 7, 3.0)   // 留学
checkbox(nbX, rowY + 11, 3.0)  // 障害者
// ← 4つ目がない
```
**修正案**: `rowY + 11` を `rowY + 9`に詰め、`rowY + 13` に「38万円以上の支払」チェックボックスを追加。
行高さ `ROW_H.rowB = 16mm` の中に 4 チェックが入る余裕を確認の上、フォントサイズを 1.8 に調整。

---

### DIFF-02（P1）: 列ヘッダー「個人番号」の補足テキストが誤り

**箇所**: `CH_Y` 行のメインテーブル列ヘッダー  
**PDF の実際の内容**:
- 「個人番号」列ヘッダー: ただ「個　人　番　号」
- 「特定扶養親族等」列ヘッダー: 「特定扶養親族等（平成15.1.2〜平成19.1.1生）」

**現在のコード**:
```typescript
lbl('個　人　番　号\n（平15.1.2〜H19.1.1生）', COL.myNumber.x, ...)  // ← 補足を間違った列に入れている
lbl('特定扶養\n年月日', COL.tokutei.x, ...)  // ← 「年月日」ではなく「親族等」が正しい
```
**修正案**:
```typescript
lbl('個　人　番　号', COL.myNumber.x, ...)
lbl('特定扶養親族等\n（平15.1.2〜\n平19.1.1生）', COL.tokutei.x, ...)
```

---

### DIFF-03（P2）: 右端「従たる給与の申告書提出」エリアが未実装

**箇所**: ヘッダー右端（「扶」スタンプ横）  
**PDF の実際の内容**:
```
┌─────────────────────────┐
│ 従たる給与についての     │
│ 扶養控除等申告書の提出   │
│ （副から受けている場合の  │
│  みここに○をつけてください）│
│       ○                  │
└─────────────────────────┘
```
**現在のコード**: 「扶」スタンプのみ。提出チェック○エリアが欠落。  
**修正案**:
```typescript
// 「扶」スタンプ右下に追記
lbl('従たる給与についての\n扶養控除等申告書の提出\n（副から受けている場合のみ\nここに○をつけてください）',
  ML + TABLE_W - 14, Y.title + 14, 14, 16, { fontSize: 1.8, textAlign: 'center' }),
checkbox(ML + TABLE_W - 9, Y.title + 28, 4),
```

### DIFF-04（P2）: 右端縦書き注意書きが未実装

**箇所**: 帳票右端に縦書きで印刷されているガイドテキスト  
**PDF の実際の内容**（右端縦書き）:
```
○この申告書は、あなたの給与について…扶養控除等を受けるために提出するものです。
○そのため…（説明文）
…申告書の「（１）申告」についての注意を読んでください。
```
**現在のコード**: 右端縦書きテキストが一切ない。  
**修正案**: `vlbl()` ヘルパーで縦書きテキストを A4 右端に配置。ただしフォント 1.8〜2.0mm では小さく視認困難なため、実用上は「帳票背景」として扱いロック要素で追加。

---

### DIFF-05（P2）: Section D の生年月日に元号選択がない

**箇所**: Section D（他の所得者が控除を受ける扶養親族等）2 行  
**PDF の実際の内容**: 「明・大・昭・令」 + 年 月 日 の入力欄  
**現在のコード**:
```typescript
lbl('生年月日', COL.myNumber.x, dRowY, ...)
input(COL.myNumber.x + 1, dRowY + 4, ...)  // ← eraSelect なし
```
**修正案**:
```typescript
eraSelect(COL.myNumber.x + 1, dRowY + 4, 7, DH/2 - 5)
input(COL.myNumber.x + 9, dRowY + 4, ...) // 年月日を続ける
```

### DIFF-06（P2）: 住民税・退職手当欄の生年月日に元号選択がない

**箇所**: 退職手当等を有する配偶者・扶養親族 行（`TY` 行）  
**PDF の実際の内容**: 「明・大・昭・令」 元号セレクタ付き生年月日  
**現在のコード**: `T_COLS` の「生年月日」列が `input()` のみ。  
**修正案**: 退職欄の生年月日列にも `eraSelect()` を先頭に追加する。

---

### DIFF-07（P2）: 住民税・16歳未満扶養親族の生年月日が「平」固定表示不明

**箇所**: 住民税 16歳未満扶養親族行（`jRowY`）の生年月日列  
**PDF の実際の内容**: 生年月日欄に「平」という小文字プレフィックスが印字されている（平成生まれ固定）  
**現在のコード**: 住民税行は `input()` のみ。「平」プレフィックスラベルがない。  
**修正案**: `lbl('平', ...)` を生年月日入力欄の左に追加し、固定の元号を示す。

---

### DIFF-08（P3）: Section C の「障害者区分」見出しが不明確

**箇所**: Section C（障害者・寡婦・ひとり親・勤労学生）の上部  
**PDF の実際の内容**: 「区分」という小見出し列があり、各行（一般・特別・同居特別）の左に「区分」ラベルがある  
**現在のコード**: 「区分」列の明示的なラベルがない。`C_COLS` で列タイトルはあるが PDF の「区分」見出しとは位置が異なる可能性。  
**修正案**: 実際の PDF の C セクション構造と照合のうえ、「区分」ラベルの位置を確認・修正。

---

### DIFF-09（P3）: 「生計を一にする事実」欄に注記テキストが欠落

**箇所**: 「生計を一にする事実」列ヘッダー下の（A 行のみ）小文字注記  
**PDF の実際の内容**: A行の「生計を一にする事実」セル内に「（該当する場合は口を付けてください）」という小文字テキストが印字されている  
**現在のコード**: `buildPersonRow` のこの列は `input()` のみ。注記テキストなし。  
**修正案**: Section A 専用に `lbl('（該当する場合は□を付けてください）', ...)` を小フォントで追加。

---

### DIFF-10（P3）: 列幅の微調整余地

**箇所**: 全体の列幅バランス  
**PDF との比較**: 実際の PDF では各列幅の比率が現テンプレートと若干異なる可能性。特に：
- `COL.myNumber.w = 22mm` → 実際は約 23mm
- `COL.address.w = 28mm` → 実際は約 30mm
- `COL.ido.w = 27mm` → 実際は約 25mm

**修正案**: Figma や PDF 寸法ツールで実際の幅を計測し定数を調整。現状は記法的に正確ではないが視覚的には近似しているため低優先。

---

### DIFF-11（P3）: 注1 テキストに脚注 2 が欠落

**箇所**: Section C の「同一生計配偶者（注2）」参照先  
**PDF の実際の内容**: 「（注2） 同一生計配偶者とは…」という説明が注1 ボックス内か別行に存在  
**現在のコード**: `Y.note1` に（注1）のみ。（注2）テキストなし。  
**修正案**: 注1 ボックスの高さを増やすか、別の `lbl()` で（注2）を追加。

## 補足: UI 操作で気づいた改善点

フルプレビュー・PNG エクスポートによる UI 確認中に気づいた操作性の課題:

### UI-01: キャンバスが A4 横幅（210mm）をウィンドウに収めきれない
- 75% ズームでも右端の「異動月日及び事由」列がサイドバーに隠れる
- 改善案: ズーム「全体に合わせる」ボタンを追加するか、デフォルトズームを `fit-to-width` に変更

### UI-02: 元号選択要素のプレビュー表示が縦に小さい
- EraSelectRenderer でのフォントサイズが `size.height / 5 * 0.75` だが、
  ヘッダー行の 5mm 要素では文字が 0.75mm になり実質不可視
- 改善案: 最低フォントサイズ（例: 2.0mm）を下限値として設定する

### UI-03: ロック要素のクリックスルーが不自由
- locked 要素がほぼ全面を覆うため、入力欄要素をクリック選択しにくい
- 改善案: Ctrl+クリックでロック要素透過選択モードを追加
