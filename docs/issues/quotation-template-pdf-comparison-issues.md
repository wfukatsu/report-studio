## 発見した差異・課題

### DIFF-Q01（P2）: 品目テーブルの空行罫線が未表示

**箇所**: 品目テーブル（10行分の空白行エリア）  
**PDF の実際の内容**: データの有無にかかわらず、全10行に水平罫線が引かれている  
**現テンプレート**: `repeatingBand` 要素のみ。データがないときは空白で罫線なし。  
**修正案**:
品目テーブル領域（`Y_BODY` 〜 `Y_BODY + TABLE_ROWS * TABLE_ROW_H`）に
10本の水平罫線を固定要素（`hline()`）として追加する。
```typescript
for (let i = 1; i <= TABLE_ROWS; i++) {
  elements.push(hline(ML, Y_BODY + i * TABLE_ROW_H, CONTENT_W))
}
```

### DIFF-Q02（P2）: 発行者ロゴ画像エリアが未実装

**箇所**: 発行者情報ブロック右上（`RIGHT_X, Y_SENDER_LOGO`）  
**PDF の実際の内容**: 会社ロゴ画像（DesignWindow ロゴ）が配置されている  
**現テンプレート**: 灰色の背景矩形 + テキスト社名のみ。画像要素なし。  
**修正案**: `image` 要素を追加し、ロゴ画像のプレースホルダーとする。
```typescript
{
  id: uuidv4(), type: 'image',
  position: { x: RIGHT_X + 2, y: Y_SENDER_LOGO + 1 },
  size: { width: 30, height: 12 },
  zIndex: 4, locked: false, visible: true,
  src: '', alt: '会社ロゴ',
}
```

### DIFF-Q03（P2）: 登録番号ラベルの位置ずれ

**箇所**: 右上情報ブロック 3行目  
**PDF の実際の内容**: 「登録番号：TXXXXXXXXXXXXX」が発行日・見積書Noと右揃え  
**現テンプレート**: ラベル位置は合っているが、`RIGHT_X` 起点から `textAlign: 'right'`
のため、ラベルとデータフィールドの横位置が揃わない可能性あり。  
**修正案**: 情報ブロック全体を `textAlign: 'left'` に統一し、ラベル幅を固定値にする。

**現テンプレート**: `LEFT_W = 106` だが、お見積金額の下線も `LEFT_W` で引かれているので概ね揃い。微調整の余地あり。  
**修正案**: 目視確認で問題なければ現状維持。

## 改善提案

### 改善-01: 計算式ルールの追加

品目データから小計・消費税・合計を自動計算するための
`CalculationRule` を定義する。バックエンド `CalculationEngine` と連携し、
以下の計算式をテンプレートに紐付ける:

```
subtotal = SUM(items[].amount)
tax10Amount = FLOOR(subtotal * 0.10)
tax8Amount = FLOOR(tax8Base * 0.08)
total = subtotal + tax10Amount + tax8Amount
```

### 改善-02: テンプレートバリアント機能

基本版・値引版・英語版を「同一テンプレートの出力バリアント」として管理し、
出力バリアント設定で切り替えられるようにする。
列の表示/非表示、ラベル言語、集計行の追加/削除を制御。

### 改善-03: 繰り返しバンドの空行罫線レンダリング

`RepeatingBandRenderer` に `showEmptyRowLines: boolean` プロパティを追加し、
データが `maxItems` 未満の場合でも残りの行に罫線を描画する機能を検討。
これにより固定要素で罫線を手動追加する必要がなくなる。
