## 1. はじめに

Report Design Studio は、請求書・見積書・帳票などの PDF テンプレートをブラウザ上でビジュアルに設計・出力するツールです。  
主な特徴:

- **ドラッグ&ドロップ**で要素を配置
- **データバインディング**でフィールドを動的に埋め込み
- **JEXL 式**による計算・集計
- **バリデーションルール**でデータ品質チェック
- **出力バリアント**で用途別 PDF を一括生成
- **バージョン履歴**で過去状態への復元

## 2. 画面構成

```
┌─────────────────────────────────────────────┐
│  ツールバー (上部)                            │
├──────────┬──────────────────────┬────────────┤
│ 左サイドバー │      キャンバス        │ 右サイドバー │
```

### 左サイドバーのタブ

| タブ | 内容 |
|------|------|
| **要素** | 配置できる要素のパレット |
| **スキーマ** | データスキーマの定義 (グループ・フィールド) |

### 右サイドバー (プロパティパネル)

要素を選択すると、その要素のプロパティ (位置・サイズ・スタイル・データバインド・条件付き表示) が表示されます。

## 3. テンプレートの作成・読み込み

### 新規作成

1. ツールバー左の **「新規」** ボタン (FilePlus アイコン) をクリック
2. 現在の内容は破棄されます (未保存の変更は事前に保存してください)

### テンプレートギャラリーから読み込み

1. ツールバーの **「テンプレート」** ボタンをクリック
2. ダイアログに組み込みテンプレートの一覧が表示されます
3. 使用するテンプレートを選択して **「このテンプレートを使用」** をクリック

### JSON ファイルから読み込み

1. ツールバーの **「開く」** ボタン (FolderOpen アイコン) をクリック
2. `.json` ファイルを選択します

### JSON として保存

1. ツールバーの **「保存」** ボタン (Save アイコン) をクリック
2. `report.json` ファイルとしてダウンロードされます

## 4. 要素の追加と配置

### パレットからの追加

1. 左サイドバーの **「要素」** タブを開く
2. 追加したい要素のボタンをクリック → ページ中央付近に配置されます
3. またはボタンを **キャンバスへドラッグ&ドロップ**

### 要素の移動

- 要素をクリックして選択し、**ドラッグ**で移動
- 矢印キーで 1mm 単位の微調整 (Shift+矢印で 5mm)

### 要素のリサイズ

- 選択中の要素の**四隅・辺のハンドル**をドラッグ

### 複数選択

- **Ctrl + クリック** (または Cmd + クリック) で複数要素を追加選択
- キャンバスの空白部分でドラッグして**範囲選択**

### コピー・切り取り・貼り付け

| 操作 | ショートカット | ツールバー |
|------|--------------|---------|
| コピー | Ctrl+C | Copy アイコン |
| 切り取り | Ctrl+X | Scissors アイコン |
| 貼り付け | Ctrl+V | Clipboard アイコン |

### 削除

選択した要素を **Delete** キーまたは **Backspace** キーで削除します。

### グリッドとスナップ

- ツールバーの **Grid アイコン** でグリッド表示 ON/OFF
- ツールバーの **Magnet アイコン** でグリッドスナップ ON/OFF

## 5. 要素の種類と設定

### テキスト系

| 要素 | 説明 |
|------|------|
| **テキスト** | 固定文字列。`{{fieldKey}}` トークンでデータを埋め込み可能 |
| **ラベル** | 印刷・サイズ固定のラベルテキスト |
| **データフィールド** | スキーマのフィールドを直接表示 (顧客名・金額など) |

**テキストへのデータ埋め込み例:**
```
請求先: {{customer.name}} 様
請求金額: {{total}} 円
```

**システム変数:**
```
ページ番号: {{$page}}
総ページ数: {{$totalPages}}
印刷日: {{$printDate}}
```

### 図形・画像

| 要素 | 説明 |
|------|------|
| **矩形** | 長方形の図形。塗り・線色・角丸を設定可能 |
| **円** | 正円・楕円 |
| **線** | 水平・垂直・斜めの直線 |
| **画像** | PNG/JPG 画像を埋め込み。データバインドで動的画像も可能 |

### 繰り返し要素

| 要素 | 説明 |
|------|------|
| **繰り返しバンド** | 配列データを行ごとに繰り返し表示 (請求書明細など) |
| **繰り返しリスト** | 配列データをカード・グリッド形式で表示 (商品カタログなど) |

繰り返し要素内のフィールドは、バインドしたスキーマグループの `dataKey` を通じて各行のデータにアクセスします。

### データ表示

| 要素 | 説明 |
|------|------|
| **テーブル** | 複数列・複数行のデータテーブル |
| **グラフ** | 棒グラフ・折れ線グラフなど |
| **QR コード** | テキストや URL を QR コードで表示 |
| **バーコード** | Code128 形式のバーコード |

### 記入欄

| 要素 | 説明 |
|------|------|
| **記入欄** | 手書き記入用のフィールド枠 (マニュアルエントリー) |

### 日本語帳票専用

| 要素 | 説明 |
|------|------|
| **印鑑** | 社印・個人印の押印欄 |
| **多段印鑑欄** | 承認フロー用の複数段印鑑ボックス |
| **収入印紙欄** | 収入印紙の貼付欄 |

## 7. データスキーマとバインディング

### スキーマの定義

1. 左サイドバーの **「スキーマ」** タブを開く
2. **「+ グループ追加」** でグループを作成
   - **マスター** — 単一レコード (顧客情報など)
   - **詳細** — 配列データ (明細行など)
3. グループ内の **「+ フィールド追加」** でフィールドを定義
   - キー (英数字・アンダースコア)
   - ラベル (表示名)
   - 型 (文字列・数値・日付・真偽値・配列・画像)

### フィールドバインディング

1. キャンバスで **データフィールド** 要素を選択
2. 右サイドバーの **「バインディング」** セクションで対象フィールドを選択

### テキスト内への埋め込み

テキスト要素のコンテンツに `{{フィールドキー}}` を記述します:

```
{{customer.name}} 様
〒{{customer.zip}} {{customer.address}}
```

ネスト:
```
{{order.items[0].product_name}}
```

### バインディングモーダル

ツールバーの **「データ」** ボタン (Database アイコン) から高度なバインディング設定モーダルを開けます。

## 8. 計算ルール (数式エディタ)

ツールバーの **「データ」→「計算」タブ** で計算ルールを管理します。

### 計算ルールの追加

1. **「+ 追加」** ボタンをクリック
2. 以下を設定:

| フィールド | 説明 | 例 |
|-----------|------|----|
| **キー** | 結果を参照するためのキー (英数字・アンダースコア) | `total_price` |
| **ラベル** | 表示名 | 合計金額 |
| **説明** | メモ (任意) | 数量×単価の合計 |
| **式** | JEXL 式 (複数行入力可) | `sum(items.map(i => i.qty * i.price))` |
| **結果型** | 数値 / 文字列 / 真偽値 | 数値 |
| **エラー時** | 0 を返す / 空にする / エラーテキスト | 0 を返す |

### JEXL 式の構文

基本的な算術・比較・論理演算子が使えます:

```
price * quantity                     // 乗算
price * (1 + tax_rate)               // 括弧・加算
amount > 0 ? "請求あり" : "請求なし"  // 三項演算子
name == "山田" && status == "active" // 比較・論理
```

### 組み込み関数

| 関数 | 説明 | 例 |
|------|------|----|
| `sum(array)` | 配列の合計値 | `sum(items.map(i => i.amount))` |
| `count(array)` | 配列の要素数 | `count(items)` |
| `round(value, places?)` | 小数点の丸め (places 省略時は整数) | `round(price * 1.1, 0)` |

### 変数参照パネル

各ルールの **「変数」** ボタンをクリックすると、参照可能な変数一覧が表示されます:

- スキーマのフィールドキー
- 他の計算ルールのキー
- 組み込み関数の一覧

変数名をクリックすると式のカーソル位置に自動挿入されます。

### テスト実行

**「テスト」** ボタンをクリックすると現在のテストデータで式を評価し、結果または エラーを表示します。

### 書式設定

結果型が **数値** または **文字列** のルールには書式を設定できます (→ [6章 数値フォーマット](#データフィールドの数値フォーマット) 参照)。

### セキュリティガード

以下の式はセキュリティ上の理由でブロックされます:
- `constructor`、`__proto__`、`prototype` を含む式
- 500 文字を超える式

### キー重複の警告

同じキーを持つルールが 2 件以上ある場合、**「キーが重複しています」** の警告が表示されます。キーはユニークにしてください。

## 9. バリデーションルール

ツールバーの **「データ」→「バリデーション」タブ** でバリデーションルールを管理します。

### バリデーションルールの追加

1. **「+ 追加」** ボタンをクリック
2. 以下を設定:

| フィールド | 説明 |
|-----------|------|
| **名前** | ルール名 |
| **条件式** | JEXL 式。`false` のときエラーとみなす |
| **メッセージ** | エラー時に表示するメッセージ |
| **重大度** | error (エラー) / warning (警告) |

### バリデーションの実行

ツールバーの **「バリデーション」** ボタン (ShieldCheck アイコン) をクリックします。

- 全ルールがパスすると緑のバッジ
- 警告・エラーがあるとオレンジ・赤のバッジ + 件数が表示

## 10. 条件付き表示

各要素は特定条件を満たす場合のみ表示/非表示にできます。

### 設定方法

1. 対象要素を選択
2. 右サイドバーの **「条件付き表示」** セクションを開く
3. **「+ 条件追加」** で条件を追加:
   - フィールドを選択
   - 演算子 (`==`, `!=`, `>`, `<`, `>=`, `<=`, `contains` など) を選択
   - 比較値を入力
4. 複数条件がある場合、**AND / OR** の結合方法を選択

### 使用例

```
// 「消費税額」テキストは tax_included == true のときのみ表示
フィールド: tax_included
演算子: ==
値: true
```

## 11. ページ管理

### ページの追加

左サイドバーの **「ページ」** タブ → **「+ ページ追加」** ボタン。

### ページの並べ替え

ページパネル内のページをドラッグして順序を変更します。

### ページ設定

ページを選択して **「ページ設定」** パネルで以下を変更できます:

| 設定項目 | 内容 |
|---------|------|
| 用紙サイズ | A4 (210×297mm) / A3 (297×420mm) / Letter (215.9×279.4mm) / Legal (215.9×355.6mm) |
| 向き | 縦 / 横 |
| 背景色 | 任意の背景色 |

### ページの複製・削除

ページパネルの各ページのコンテキストメニューから実行します。

## 12. ヘッダー・フッター

全ページに共通のヘッダー・フッターを設定できます。

### ヘッダー編集モードへの切り替え

ツールバーの **「ヘッダー」** ボタン (PanelTop アイコン) をクリックして編集モードを切り替えます。

- ヘッダー編集モードでは、ヘッダー領域が黄色くハイライトされます
- 通常と同じ操作で要素を追加・編集できます
- ヘッダーを閉じるには再度ボタンをクリック

ヘッダーの高さは要素を配置した最大 Y 位置に自動調整されます。

## 13. 出力バリアント (PDF 書き出し設定)

用途別に異なる PDF を生成するための設定です。例: 「社内用」「顧客用」「控え」など。

### バリアントの管理

ツールバーの **「バリアント」** ボタン (Shuffle アイコン) をクリック。

#### バリアントの追加

1. **「+ バリアント追加」** をクリック
2. 名前と説明を設定
3. **含めるページ** を選択 (ページの包含/除外)
4. **要素の表示/非表示オーバーライド** を設定 (特定要素をこのバリアントのみ表示/非表示)

### PDF 書き出し (バリアント指定)

ツールバーの **「PDF 書き出し」** ボタン横の下矢印から出力バリアントを選択して書き出します。

---

## 14. プレビューと PDF 書き出し

### プレビューモード

ツールバーの **目のアイコン** をクリックしてプレビューモードに切り替えます。

- データバインドが適用された状態でレンダリングされます
- 要素の選択・編集はできません
- 再度アイコンをクリックで編集モードに戻ります

### ライブプレビュー

ツールバーの **ライブプレビューボタン** を ON にすると、編集中もプレビューが自動更新されます。

### PNG 書き出し

1. ツールバーの **「PNG 書き出し」** ボタン (FileImage アイコン) をクリック
2. 現在のアクティブページが PNG としてダウンロードされます

### PDF 書き出し

1. ツールバーの **「PDF 書き出し」** ボタン (FileText アイコン) をクリック
2. 全ページが 1 つの PDF としてダウンロードされます

---

## 15. フォーム回答の管理

フォーム回答データを収集・管理する機能です。

### 回答の閲覧

左サイドバー下部の **「回答」** タブ (ResponsesPanel) で一覧が表示されます。

### 回答の送信 (SubmitResponseModal)

ツールバーまたはアプリ内のボタンから回答送信モーダルを開き、テンプレートに対してフォームデータを登録します。

### 集計・エクスポート

バックエンド API を通じて:
- `GET /api/v2/templates/:id/responses` — 回答一覧取得
- `GET /api/v2/templates/:id/responses/export` — CSV/Excel エクスポート

---

## 16. バージョン履歴

テンプレートの過去バージョンを閲覧・復元できます。

### バージョン一覧の表示

左サイドバーの **「バージョン履歴」** タブを開きます。

- 保存日時・バージョン番号が一覧表示されます

### バージョンの復元

1. 復元したいバージョンを選択
2. **「このバージョンに戻す」** をクリック
3. 現在の内容はそのバージョンで上書きされます

## 20. トラブルシューティング

### 要素が動かない

- **プレビューモード** になっていないか確認 (ツールバーの目アイコン)
- グリッドスナップが有効の場合、意図しない位置にスナップすることがあります

### データが表示されない

- スキーマにフィールドが定義されているか確認
- `{{fieldKey}}` のスペルがスキーマのキーと一致しているか確認
- テストデータ (DataSource) が設定されているか確認

### 計算式がエラーになる

- `テスト` ボタンで式を評価してエラー内容を確認
- キーが英数字・アンダースコアのみであるか確認 (日本語キー不可)
- `sum()`、`count()`、`round()` 以外の関数は使えません
- 式が 500 文字を超えていないか確認

### PDF 書き出しに失敗する

- バックエンドが起動しているか確認 (`npm run dev:backend`)
- バックエンドログでエラーを確認
- ページ数・要素数が多い場合は時間がかかることがあります

### ページが正しく印刷されない

- 用紙サイズと向きが正しく設定されているか確認 (左サイドバー「ページ」タブ)
- トリムマーク (トンボ) を ON にしてから PDF 出力することを推奨します
