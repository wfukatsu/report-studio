# UI/UX デザイナー視点の評価

## 概要

Report Design Studio v2 の UI を デザイナー視点で全体評価した結果。
既存の UI は「よく整理された帳票エディタ」として機能しているが、
**一貫性・発見可能性・状態の明確さ** に改善の余地がある。

---

## 強み（維持すべき点）

- **3 パネル構成**（左サイドバー・キャンバス・右サイドバー）は DTP ツールの標準的なレイアウトで直感的
- **共通 UI プリミティブ**（`PropSection` / `PropRow` / `NumInput` など）で各プロパティ編集UIの見た目が統一されている
- **dnd-kit** による滑らかな DnD、8 ハンドルリサイズなど編集操作は充実
- **キーボードショートカット**（⌘Z/C/X/V/D/A）は標準的で学習コストが低い
- **エラーバウンダリ・バリデーション** など安全網が実装されている

---

## 課題カテゴリ（優先度順）

### 🔴 P1 — ユーザーが詰まる・混乱する

#### 1. ズーム状態の分裂
- **症状**: エディタズームとプレビューズームが別管理。不一致時に「—」表示になるが意味が分かりにくい
- **影響**: 初見ユーザーが「なぜズームが効かないのか」と混乱する
- **方針案**: デフォルトは同期、独立設定は上級機能として隠す or 廃止検討

#### 2. 貼り付け位置の不明確さ
- **症状**: ペースト後の要素の配置先がユーザーに伝わらない（コード上はボディ中央付近だが表示なし）
- **影響**: 「どこに貼った？」とキャンバスをスクロールして探す羽目になる
- **方針案**: ペースト後に対象要素を選択状態にし、スクロールして見える位置に移動する（既に選択はしているか要確認）

#### 3. マスターヘッダー/フッターの編集フローが不明
- **症状**: ツールバーの「H」「F」ボタンと「H/F編集」モードの関係が不明瞭
- **影響**: セクションを編集しているのか、サイズ変更しているのか区別がつかない
- **方針案**: ヘッダー/フッター編集モード中はキャンバス上部/下部に明示的なバナーを表示し、モード状態を強調

---

### 🟡 P2 — 発見しづらい・一貫性が低い

#### 4. ボタンスタイルの不統一
- **症状**: ツールバーで「アイコンのみ」と「アイコン＋テキスト」ボタンが混在。無効化の透明度も場所によって 30%/40% とまちまち
- **方針案**: デザイントークンとして `--opacity-disabled: 0.35` を統一定義。アイコンのみは補助操作、テキスト付きは主要操作と明確にルール化

#### 5. フォーカス/アクティブ状態の不統一
- **症状**: ツールバーアクティブは「青背景塗りつぶし」、タブのアクティブは「下線」、プロパティ入力は「リング」とバラバラ
- **方針案**: 各 UI 要素種別ごとにアクティブ表現を統一（インタラクティブ UI ガイドライン策定）

#### 6. バリデーションの発見可能性
- **症状**: バリデーションルールは DataBindingModal の奥にあり、プロパティパネルからは直接アクセスできない
- **方針案**: プロパティパネルに「バリデーションルールを追加」ショートカットリンクを設置

#### 7. パレット要素のdescriptionが見えない
- **症状**: 各パレットアイテムに `description` フィールドがあるが、`title` 属性のみでホバー時に OS デフォルトのツールチップにしか見えない
- **方針案**: カスタムツールチップ（ポップオーバー形式）で説明と使用例を表示

#### 8. バージョン履歴でアンドゥプレビューがない
- **症状**: 履歴エントリをクリックすると確認なしに即ロールバックされる
- **方針案**: 確認ダイアログを挟む or エントリにホバーするとキャンバスにプレビューオーバーレイを表示

---

### 🔵 P3 — あれば嬉しい・将来的な改善

#### 9. テーマ切替ボタンの欠如
- CSS 変数でダークモード対応済みだが、UI からトグルできない

#### 10. モバイル/タブレット非対応
- 3 パネルレイアウトは小画面で破綻する
- タブレット向けに「サイドバー折りたたみ」モードがあれば実用的

#### 11. アクセシビリティの不完全な実装
- 一部ボタンで `title` はあるが `aria-label` が欠けている
- ドラッグ選択（マーキー）にキーボード代替がない
- カラーのみの状態表示（未保存ドット等）にテキスト代替が必要

#### 12. ドラッグ&ドロップの視覚フィードバック不足
- パレットからキャンバスへのドラッグ中、ドロップ可能ゾーンのハイライトがない

---

## 改善の進め方（推奨アプローチ）

### Phase 1: 即効性の高い修正（P1）
各々独立した小さな変更。リスク低・効果大。

1. ズーム同期をデフォルト化（`setZoom` が両方更新する形に変更）
2. ペースト後のオートスクロール＆選択維持の確認
3. ヘッダー/フッター編集モードのバナー表示

### Phase 2: デザインシステム整備（P2前半）
コード規模は大きいが、一括で直すと一貫性が劇的に向上。

4. disabled 透明度トークン統一
5. アクティブ状態の統一（ツールバー/タブ）

### Phase 3: 機能改善（P2後半・P3）
新機能の追加を伴うため計画が必要。

6. バリデーションショートカット
7. カスタムツールチップ
8. バージョン履歴の確認ダイアログ

---

## 解決済みの決定事項

1. **ズーム**: エディタとプレビューの独立ズームは**意図的な機能として維持**する。
   ただし「—」表示を廃止し「エディタ 100% / プレビュー 75%」のように両方の値を明示表示する。
2. **優先度**: **P1 から順番に**着手する。
