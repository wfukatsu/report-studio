# テンプレート分類機能 ブレインストーム

**日付:** 2026-04-10
**ステータス:** 承認済み

---

## 何を作るか

帳票テンプレートにカテゴリ（ユーザー管理の大分類）とタグ（自由ラベル）を付与し、テンプレート選択モーダルでフィルタチップ＋検索バーによる絞り込みを可能にする。

ビルトインテンプレートとバックエンドテンプレートの両方に適用する。

## なぜこのアプローチか

**アプローチA**: `Template` 型と `ReportDefinition.metadata` に `category` / `tags` フィールドを追加。

- テンプレートデータと一体で管理するため、JSON export/import でも分類が保持される
- 既存の保存/読込パイプラインに自然に乗る（Zod schema は `passthrough()` で未知フィールドを許容済み）
- 別ストア方式（B）やlocalStorage方式（C）に比べてデータの一貫性が高い

## キー決定事項

### 1. 分類構造: カテゴリ＋タグの併用

- **カテゴリ**: 単一選択の大分類（例: 「税務」「請求・見積」「社内文書」）。テキスト入力で自由に追加可能。未使用カテゴリは自動消滅
- **タグ**: 複数付与可能な自由ラベル（例: 「英語」「A4」「承認付き」）。ユーザーが自由に作成

### 2. データモデル

```ts
// Template 型に追加
interface Template {
  // 既存フィールド...
  category?: string        // カテゴリ名（例: "税務"）
  tags?: string[]           // タグ配列（例: ["英語", "A4"]）
}

// ReportDefinition.metadata に追加
interface Metadata {
  // 既存フィールド...
  category?: string
  tags?: string[]
}

// TemplateListItem (API) に追加
interface TemplateListItem {
  // 既存フィールド...
  category?: string
  tags?: string[]
}
```

### 3. カテゴリ/タグの付与UI

テンプレートへのカテゴリ/タグ設定は以下の2箇所で行う:

- **ページ設定パネル（右サイドバー）**: 現在の `metadata` を編集するUIの延長として、カテゴリ選択ドロップダウンとタグ入力欄を追加
- **テンプレート保存時**: 保存ダイアログにカテゴリ/タグ入力欄を表示

### 4. カテゴリ一覧の管理

- カテゴリ一覧は全テンプレートの `category` フィールドから動的に集約する（マスタテーブル不要）
- 新しいカテゴリはカテゴリ選択UIでテキスト入力して追加する
- カテゴリの「削除」は、全テンプレートからそのカテゴリを外せば自動的に消える
- ビルトインテンプレートには初期カテゴリをプリセット

### 5. UI: フィルタチップ＋検索バー

```
┌─────────────────────────────────────────┐
│ [🔍 テンプレートを検索...              ] │
│ [すべて] [税務] [請求・見積] [社内] +   │
│─────────────────────────────────────────│
│ タグ: [英語] [A4] [承認付き]        ×   │
│─────────────────────────────────────────│
│ [テンプレートA] [テンプレートB]          │
│ [テンプレートC] [テンプレートD]          │
└─────────────────────────────────────────┘
```

- 上段: テキスト検索バー（名前・説明で検索）
- 中段: カテゴリチップ（排他選択、「すべて」で全表示）
- 下段: タグチップ（複数選択、AND絞り込み）
- `+` ボタンでカテゴリ追加

### 6. 適用範囲

- ビルトインテンプレート: `BUILTIN_TEMPLATES` 配列の各テンプレートに `category` / `tags` をプリセット
- バックエンドテンプレート: `metadata.category` / `metadata.tags` として保存・読込
