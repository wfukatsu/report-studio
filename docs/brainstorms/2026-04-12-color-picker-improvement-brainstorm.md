# カラーピッカー改善

## What We're Building

カラーピッカーを「ブランドカラー登録パレット＋最近使った色の自動追跡」に刷新する。
現在の `<input type="color">` をポップオーバー型 UI に置き換え、
ワンクリックでブランドカラーを選択できるようにする。

対象: `src/elements/_base/sharedUI.tsx` の `ColorInput` コンポーネント

## Why This Approach

**困り事:** 会社・ブランドカラー（例: #E74C3C, #2C3E50）を複数の要素に毎回手入力している。  
**解決策 C** = ブランドカラーを localStorage に永続化（ A）+ 最近使った色を自動記録（B）。

- A だけ: 最初の一回登録すれば全レポートで使い回せる。テンプレートに色定義を埋め込みたい場合も将来拡張できる。
- B も加える: 登録を忘れても操作の流れで自然に蓄積され、二度目以降はワンクリック。
- localStorage 保存: ユーザー環境に紐づくため、レポートが変わっても引き継がれる。

## ポップオーバー UI 設計

```
文字色  [■ #E74C3C ▼]
         ┌─────────────────────────┐
         │ ブランドカラー   [管理] │
         │ ■ ■ ■ ■ ■ ■            │
         │─────────────────────────│
         │ 最近使った色            │
         │ ■ ■ ■ ■ ■ ■            │
         │─────────────────────────│
         │ カスタム: [#______] ✓  │
         └─────────────────────────┘
```

- **ブランドカラー行**: localStorage に保存されたプリセット。最大 12 色程度。
- **最近使った色**: 色を確定するたびに自動追加（最大 8 件、重複排除）。localStorage に保存。
- **カスタム入力**: HEX コードを手打ちできるフォールバック。現行機能を継続。
- **[管理] リンク**: ブランドカラーの追加・削除・並べ替えができる小モーダルを開く。

## localStorage スキーマ（案）

```ts
// key: "rds:brandColors"
type BrandColor = { hex: string; name?: string }
type BrandColorsStore = BrandColor[]  // 最大 12 件

// key: "rds:recentColors"
type RecentColorsStore = string[]  // HEX 文字列, 最大 8 件
```

## Key Decisions

- **ポップオーバー型 UI**: ブランドカラー・最近の色・カスタム入力を 1 画面に集約
- **localStorage 保存**: レポートを跨いで引き継がれるユーザー設定として管理
- **ColorInput を置き換え**: 既存の `sharedUI.tsx` の `ColorInput` を拡張してポップオーバー化
- **最大件数**: ブランドカラー 12 件・最近の色 8 件（YAGNI: 設定可能にしない）
- **ブランドカラーに名前あり**: ホバー時にツールチップで名前を表示
- **管理 UI は別モーダル**: ポップオーバーの「[管理]」リンクから開く独立モーダル
- **最近の色は確定時のみ追加**: ホバー中は追加しない（ノイズ防止）

## Resolved Questions

1. ✅ **名前あり** — スウォッチにホバーで名前をツールチップ表示
2. ✅ **別モーダル** — ポップオーバー内の「管理」ボタンで開くモーダル（追加・削除・名前編集）
3. ✅ **確定時のみ** — 色確定（Enter / フォーカスアウト）時に最近の色に追加
4. ✅ **初期データはデフォルト彩色を用意** — localStorage が空のとき汎用的なデフォルト色セットを表示し、登録ハードルを下げる
5. ✅ **単一パレットのみ** — 複数パレット切り替えは不要（YAGNI）
