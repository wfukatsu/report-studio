# Brainstorm: データブラウザ機能

**Date:** 2026-04-12
**Status:** Draft
**Feature:** 複数データソースを横断的に閲覧できるデータブラウザ専用ページ

---

## What We're Building

帳票デザイナーとは独立した専用ページとして、以下3種類のデータを横断的に閲覧・検索・エクスポートできるデータブラウザを追加する。

- **ScalarDB テーブルデータ** — データ連携タブで接続済みのテーブルを行単位で閲覧
- **商品マスター** — 登録済み商品をグリッドで閲覧・絞り込み
- **フォーム回答データ** — 帳票ごとの送信済み回答を一覧表示

### レイアウト

```
+--[左サイドバー: ツリー]--+--[右ペイン: データグリッド]--+
| ▶ ScalarDB               | [検索バー] [フィルタ] [CSV]   |
|   └ orders テーブル       | ┌─────────────────────────┐   |
|   └ customers テーブル    | │ col1 | col2 | col3 | ... │   |
| ▶ 商品マスター            | │ 行データ...               │   |
| ▶ フォーム回答            | └─────────────────────────┘   |
|   └ 帳票A                 | [< 前ページ] 1/5 [次ページ >] |
|   └ 帳票B                 |                               |
+--------------------------+-------------------------------+
```

## Key Decisions

### 1. ルーティング

- 専用ページとして `/data-browser` に配置（帳票デザイナーとは完全分離）
- トップバーに「データブラウザ」リンクを追加
- **React Router** でルートを追加（既存のルーティング方式に準拠）

### 2. 左サイドバー: データソースツリー

| ノード | 説明 |
|--------|------|
| `ScalarDB` (折りたたみ可) | ScalarDB カタログ API から取得したテーブル一覧 |
| `商品マスター` | 固定ノード（productSlice から取得） |
| `フォーム回答` (折りたたみ可) | テンプレート一覧から動的生成 |

### 3. 右ペイン: データグリッド

**共通機能（全データソース共通）:**
- 検索バー（**クライアントサイドフィルタ** — 取得済みの50件/ページ内を絞り込む）
- ソート（列ヘッダークリック、クライアントサイド）
- ページネーション（50件/ページ）
- CSV エクスポート（表示中データのみ）
- 行クリックで詳細パネル（**右スライドインパネル** — 一覧を見ながら詳細を確認できる）

**データソース別の差異:**

| データソース | データ取得方法 | 列の定義 |
|------------|--------------|---------|
| ScalarDB テーブル | 新規 scan API `GET /api/v2/scalardb/tables/{ns}/{table}/rows` | ScalarDB の `TableMetadata` から動的生成 |
| 商品マスター | `GET /api/v1/products`（既存） | Product 型の固定フィールド |
| フォーム回答 | `GET /api/v2/templates/{id}/responses`（既存） | 回答フィールドから動的生成 |

### 4. ScalarDB データ取得方式

**課題:** 現在の `resolve-bindings` はテンプレートに紐づき、partition key が必要。データブラウザには「全件スキャン」が必要。

**選択肢:**
- **A（推奨）: 新規 scan エンドポイント** `GET /api/v2/scalardb/tables/{ns}/{table}/rows` を追加
  - ページネーション・フィルタ対応
  - 認証済みユーザーのみアクセス可
- **B: 既存 resolve-bindings を流用** — partition key なしでの Scan を許可するよう拡張（セキュリティリスクあり）

→ **Option A を採用**（セキュリティ境界を明確に保つ）

### 5. CSV エクスポート

- 表示中のページデータのみをエクスポート（全件はフェーズ2以降）
- ブラウザサイドで CSV 生成（既存の `downloadBlob()` を流用）

### 6. 詳細表示（ドリルダウン）

- 行クリックで右スライドインパネルを表示
- 全フィールドを key-value 形式で表示
- 商品マスターの場合は価格履歴も表示

---

## Why This Approach

1. **専用ページ（ルートレベル）**: 帳票デザイナーとの関心分離が明確。URLを直接共有できる。
2. **左サイドバーツリー**: 複数データソースを一覧でき、どのデータを見ているかが常に明確。
3. **既存APIの最大活用**: 商品マスター・フォーム回答は既存APIをそのまま使用してスコープを最小化。
4. **ScalarDB のみ新規 API**: 全件スキャンはセキュリティ上の新規エンドポイントが必要なため分離。
