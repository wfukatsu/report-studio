**Date:** 2026-04-06

## What We're Building

LayersPanelの右クリックコンテキストメニュー。既存のキャンバス用`ContextMenu`コンポーネントを`items`プロップで設定可能に拡張し、LayersPanel固有の操作（グループ化・グループ管理・要素管理）を追加する。

メニューは右クリック対象によって**2種類のコンテキスト**を持つ：

### A) 要素行を右クリック

```
コピー          ⌘C
カット          ⌘X
ペースト        ⌘V
複製            ⌘D
名前変更        F2
─────────────────
グループ化      ⌘G   ← 複数選択時のみ有効
グループから解除     ← グループ内要素時のみ表示
─────────────────
最前面へ
前面へ
背面へ
最背面へ
─────────────────
表示 / 非表示
ロック / 解除
─────────────────
削除            ⌫
```

### B) グループ行を右クリック（前ブレインストームで設計）

```
グループ名変更  F2
新規グループ追加
─────────────────
グループ解除
─────────────────
表示 / 非表示  ← グループ全体
ロック / 解除  ← グループ全体
─────────────────
グループを削除
```

## Why This Approach

### アプローチ A: 既存ContextMenuを拡張・共通化（推奨）

既存の`ContextMenu`コンポーネントに`items`プロップ（メニュー項目の配列）を追加して設定可能にする。キャンバスとLayersPanelで同じコンポーネントを使い、表示項目だけを変える。

**Pros:**
- コンポーネント重複なし（キーボードナビ・位置調整ロジックを一元管理）
- テストが一カ所で済む
- 既存のアクセシビリティ実装（`role="menu"`, `ArrowUp/Down`）を再利用

**Cons:**
- `ContextMenu`コンポーネントの既存インターフェースを変更する必要がある（破壊的変更だが影響箇所は限定的）

### アプローチ B: LayersPanel専用に別コンポーネントを作成

`LayerContextMenu.tsx`として独立したコンポーネントを新規作成する。

**Pros:**
- 既存コードへの影響ゼロ

**Cons:**
- キーボードナビ・クリック外検知などのロジックが重複
- 将来的に両方をメンテナンスするコストが増える

### アプローチ C: `MenuItem`だけ共通化、コンテナは別

`MenuItem`関数コンポーネントをエクスポートして再利用し、コンテナ部分はLayersPanel専用に書く。

**中間案**だが中途半端。アプローチ Aの方が一貫性が高い。

**→ アプローチ A を採用。**

## Key Decisions

| 決定事項 | 選択 | 理由 |
|---------|------|------|
| 既存ContextMenuとの関係 | 共通化（items propsで設定可能に） | DRY、アクセシビリティの一元管理 |
| メニューの種類 | 2種類（要素用 / グループ用） | 右クリック対象によってコンテキスト切り替え |
| グループ化メニュー項目 | 複数選択時のみ有効（disabled制御） | 1要素選択時はグレーアウト |
| グループ解除 | グループ内要素のみ表示 | グループ外要素には不要 |
| 名前変更 | F2キー or メニューから起動 | 既存のダブルクリックと共存 |
| 新規グループ追加 | グループ行右クリックメニューから | 空グループ作成 → その後要素をassign |
| グループ削除時の挙動 | 確認ダイアログで選択 | 「要素も削除」か「グループ解散のみ」をユーザーが決める |
| ⌘G スコープ | キャンバス・LayersPanel両方 | 選択中の要素があればどちらからでもグループ化可能 |
| グループ解除のUI | 要素行=個別脱退、グループ行=全解散 | 粒度の異なる2操作として両方提供 |

## キャンバスContextMenuとの差分

| 操作 | キャンバス | LayersPanel要素 | LayersPanelグループ |
|------|-----------|-----------------|---------------------|
| コピー/カット/ペースト/複製 | ✅ | ✅ | ✗（グループ構造のコピーは対象外） |
| Z順序変更 | ✅ | ✅ | ✗（グループはzIndex概念なし） |
| 表示/ロック | ✅ | ✅ | ✅（グループ一括） |
| 削除 | ✅ | ✅ | ✅（グループ解散 or グループ+要素全削除） |
| グループ化（右クリックメニュー） | ✗ | ✅ | ✗ |
| グループ化（⌘G ショートカット） | ✅ | ✅ | ✗ |
| グループ解除 | ✗ | 条件付き（グループ内要素のみ） | ✅ |
| 名前変更 | ✗ | ✅ | ✅ |
| 新規グループ追加 | ✗ | ✗ | ✅ |

## Resolved Questions

1. **グループ削除時の挙動** → 確認ダイアログ「グループ内の要素も削除しますか？」を表示。ユーザーが選択できる（削除 or グループ解散して要素は残す）。
2. **グループ化のショートカット（⌘G）** → キャンバスで要素を選択中でも有効。LayersPanel・キャンバス両方でグループ化できる。
3. **グループ解除の表示場所** → 両方に出す。要素行右クリック:「グループから外す（個別脱退）」/ グループ行右クリック:「グループ解散（全要素をセクションルートへ）」。
