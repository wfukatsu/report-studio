# FormTable Excel風テーブル編集ブレインストーム

**日付:** 2026-04-09
**ステータス:** approved
**次のステップ:** `/workflows:plan` で実装計画を策定

## What We're Building

formTable要素にExcelのようなインタラクティブ編集機能を追加する。キャンバス上でテーブルを直接操作できるようにし、既存のPropertiesPanelと併存する形で直感的な編集体験を提供する。

### 4つの主要機能

1. **キャンバス上でのセル直接編集** — ダブルクリックでテーブル編集モードに入り、セル選択・ポップオーバーでフルインライン編集（テキスト・セルタイプ・フィールドキー・スタイルすべて）
2. **行・列の追加/削除/並べ替え** — 右クリックメニューまたはヘッダー操作で行列を構造的に操作
3. **セル結合・分割** — 複数セルをドラッグ選択し、テーブル編集ツールバーの結合ボタンで colspan/rowspan
4. **列幅・行高のドラッグリサイズ** — 境界線をドラッグして視覚的にサイズ調整

## Why This Approach

**アプローチ: Renderer拡張 + インタラクティブオーバーレイ**

既存の `FormTableRenderer` のHTML table描画の上にインタラクション層を追加する。

理由:
- 既存コードとの差分が最小
- HTML tableの `colspan`/`rowspan` がそのまま使える
- 印刷/PDF出力との表示一貫性を保てる
- 既存テンプレートのマイグレーション不要

却下したアプローチ:
- **仮想グリッドエンジン**: 2Dグリッドで再構築。柔軟だが大規模改修+マイグレーション必要
- **外部ライブラリ統合**: AG Grid等。帳票特有要件（mm単位、印刷最適化）との統合困難

## Key Decisions

### 1. テーブル編集モードの導入（操作フロー）

3段階のモード遷移:

1. **通常モード** — クリックでテーブル全体選択（dnd-kitによる移動/リサイズ）
2. **テーブル編集モード** — ダブルクリックで進入。クリックしたセルが選択状態になる
   - クリック → 別のセルを選択
   - Shift+クリック / ドラッグ → 範囲選択（結合操作用）
   - Tab/矢印キー → セル間移動
3. **セル編集** — テーブル編集モード中にセルをダブルクリック（またはEnter）でポップオーバー編集開始

モード解除: Esc でセル編集→テーブル編集モード→通常モードへ段階的に戻る。テーブル外クリックで即座に通常モードへ。

**テーブル編集ツールバー**: テーブル編集モード進入時にテーブル上部にフローティングツールバーを表示。結合/分割ボタン、行列追加ボタンなどを配置。

**dnd-kitとの競合回避**: テーブル編集モード中は `useDraggable` を無効化し、クリック/ドラッグをセル操作に振り向ける。通常モードに戻ると `useDraggable` が再有効化される。

### 2. セルインライン編集（ポップオーバー）

セル編集モードでポップオーバーUIを表示し、以下を編集可能:
- セルタイプ（label / input / dataField）
- テキスト内容 / フィールドキー
- フォーマット設定
- スタイル（フォント、配置、色）

### 3. セル結合のデータモデル

`FormTableCell` に `colspan`、`rowspan`、`mergedInto` を追加:

```ts
interface FormTableCell {
  id: string
  type: FormTableCellType
  // ... existing fields
  colspan?: number     // デフォルト 1
  rowspan?: number     // デフォルト 1
  mergedInto?: string  // 結合先セルのid（このセルが非表示であることを示す）
}
```

`mergedInto` は結合先セルのidを保持する。描画時に `mergedInto` が設定されたセルはスキップする。`merged: boolean` ではなく `mergedInto: string` にすることで、結合解除時にどのセルグループに属するか追跡できる。

**エッジケース — 結合セルを含む行/列の削除:**
- rowspan/colspanにまたがる行/列を削除する場合、結合を自動解除してから削除する
- 結合の起点セルが含まれる行/列を削除する場合も同様に自動解除

### 4. 列幅・行高リサイズ

- 列ヘッダーの境界線上にリサイズハンドル（カーソル変更）
- ドラッグ中はガイドライン表示
- mm単位でスナップ（最小3mm制約を維持）

### 5. 行・列操作

- テーブル編集モード中の右クリックメニューで操作:
  - 行: 上に挿入 / 下に挿入 / 削除 / 上に移動 / 下に移動
  - 列: 左に挿入 / 右に挿入 / 削除 / 左に移動 / 右に移動
- 行・列ヘッダー（端）をクリックで行/列全体選択

## Resolved Questions

- **フェーズ分けするか?** → 全機能一括実装
- **セル結合のUI?** → Excel風の範囲選択+結合ボタン
- **インライン編集の範囲?** → フルインライン（タイプ・テキスト・フィールドキー・スタイルすべて）
- **モード切替?** → ダブルクリックでテーブル編集モード、Escで戻る
- **キーボードナビゲーション?** → 実装する。Tab/矢印キーでセル間移動、Enterで編集開始
- **Undo/Redo管理?** → テーブル専用undo stack。モード内では細かい操作単位で戻せる。テーブル編集モードを抜けると全操作が1つのstore-level undoエントリにまとまる
- **コピー&ペースト?** → 対応する。内部コピー（Ctrl+C/V）に加え、Excel/スプレッドシートからのタブ区切りテキスト貼り付けにも対応
- **PropertiesPanelとの関係?** → 併存。キャンバス上でもPropertiesPanelでも編集可能。既存ユーザーの操作を壊さない
