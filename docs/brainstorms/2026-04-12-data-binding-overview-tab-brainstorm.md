# Brainstorm: データバインディング見通しタブ

**Date:** 2026-04-12  
**Status:** Ready for planning

---

## What We're Building

左サイドバーに **「データ」タブ** を追加し、レポート内のデータバインディング状況を一覧できるパネルを実装する。

現在は要素ごとにプロパティパネルを開かないとバインディング設定が確認できないため、「どの要素が未設定か」「どのフィールドがどこで使われているか」が把握しにくい。この問題をサイドバーの専用タブで解決する。

---

## Why This Approach

- v1 の `BindingMapper` コンポーネントが同様の課題を解決しており、UIパターンとして実績がある
- 左サイドバーへのタブ追加は既存アーキテクチャ（`ElementPalette` / `TemplateGallery`）と一貫性を保てる
- シンプルなリスト形式はフローティングパネルや2ペインよりも実装コストが低く、YAGNI 原則に合致する

---

## Key Decisions

### 1. UIレイアウト

左サイドバーに 3 番目のタブ「データ」を追加。タブ切り替えで既存の「要素」「テンプレート」と共存。

```
[要素] [テンプレート] [データ]
```

### 2. パネル構成（3セクション）

**セクション1: 未バインド要素 ⚠**
- 以下の2種類を「未バインド」と判定して一覧表示:
  - `text` 要素: テキスト内に `{{fieldKey}}` トークンが1つも含まれていないもの
  - `dataField` / `checkbox` / `eraSelect` 等: `fieldKey` が未設定のもの
- 件数バッジ付き（0件なら非表示）
- 各行をクリックすると該当要素をキャンバス上で選択

**セクション2: フィールドマッピング ✓**
- `fieldKey → 要素名` の対応を一覧表示
- DataSource のフィールドと要素の紐付けをリストで確認できる
- 使用されていないフィールドは薄いテキストで表示

**セクション3: バインディングエラー ✖**
- DataSource に存在しないフィールドキーを参照している要素を一覧
- クリックで要素選択
- 0件なら非表示

### 3. データ収集ロジック

- ストアの全ページ・全セクション・全要素を走査して「bindable な要素」を抽出
- DataSource の fields（フラット化したキー一覧）と突き合わせてステータスを判定
- `useMemo` でメモ化し、要素/データ変更時のみ再計算

### 4. v1 参照

- `BindingMapper.tsx` のステータスバッジ・アイコン表示を参考にする
- `FieldCard.tsx` のバインディングドット（●/○）インジケーターを参考にする
- 実装はゼロから書くが、v1 のビジュアル言語（✓/⚠/✖アイコン、色）を踏襲する

---

## Scope (In / Out)

| In Scope | Out of Scope |
|----------|-------------|
| 未バインド要素リスト | ドラッグ&ドロップでバインド設定 |
| フィールド→要素マッピング一覧 | 複数DataSource対応 |
| バインディングエラー一覧 | フィールドの型チェック |
| 行クリックで要素選択 | データアップデート差分通知 |
| ページ横断の集計 | DB/API接続 |

## Resolved Questions

- **配置場所**: 左サイドバーのタブ追加（フローティングパネル・右サイドパネルは不採用）
- **表示形式**: シンプルリスト（タブ式2ペイン・ツリー形式は不採用）
- **対象スコープ**: 未バインド要素 / フィールドマッピング / エラーの3セクション
- **v1参照**: BindingMapper・FieldCard のビジュアル言語を踏襲
- **未バインドの定義**: text要素（{{}}トークンなし）と dataField系（fieldKey未設定）の両方を対象
- **0件時の表示**: セクションを非表示にする
- **DataSource未設定時**: 「データソースが未設定です」メッセージ + DataSourcePanel へのリンクを表示
