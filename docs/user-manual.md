# ユーザーマニュアル

**システム名:** Report Design Studio V2  
**作成日:** 2026-04-12

---

## はじめに

Report Design Studio V2 は、ブラウザ上でドラッグ&ドロップを使って帳票・フォームをデザインできる Web アプリケーションです。PDF や PNG での出力、データバインディングによる動的な帳票生成など、ビジネス帳票に必要な機能を提供します。

### 動作環境

- **ブラウザ:** Google Chrome (最新版推奨)、Edge、Safari
- **URL:** `http://localhost:5173`

---

## 1. ログイン

アプリを開くとログイン画面が表示されます。

1. **ユーザーID** に `admin` を入力
2. **パスワード** に `changeme` を入力
3. **ログイン** ボタンをクリック

> **注意:** ログイン試行は5分間に5回までです。上限を超えるとしばらく待つ必要があります。

ログインに成功すると、帳票デザイナー画面に移動します。

## 3. 新規帳票の作成

### 3.1 白紙から作成

1. ツールバー左端の **「新規作成」** ボタンをクリック
2. 白紙の帳票が表示されます

### 3.2 テンプレートから開始

1. ツールバーの **「テンプレート管理」** ボタンをクリック
2. **ビルトインテンプレート** の一覧から使いたいテンプレートを選択
3. テンプレートが読み込まれます

**利用可能なビルトインテンプレート:**
- 見積書（インボイス対応）
- 見積書（値引対応・インボイス対応）
- Quotation (English)
- 扶養控除等申告書（令和7年分）
- 要素ショーケース（全要素の確認用）

### 3.3 ページ設定

右サイドバーの **「ページ設定」** タブで用紙を設定します。

| 設定 | 説明 |
|------|------|
| ページ名 | このページの名前 |
| 用紙サイズ | A4、A3、B5、Letter など |
| 方向 | 縦 (Portrait) / 横 (Landscape) |
| 余白 | 上下左右の余白 (mm) |
| 余白プリセット | 通常 / 狭い / 広い |
| 背景色 | ページの背景色 |
| カテゴリ・タグ | テンプレート管理用のメタデータ |
| ヘッダー高さ | ヘッダーセクションの高さ (mm) |
| フッター高さ | フッターセクションの高さ (mm) |

## 4. 要素の追加と編集

### 4.1 要素の追加 (ドラッグ&ドロップ)

1. 左サイドバーの **「要素」** タブを開く
2. 追加したい要素をドラッグ
3. キャンバス上の目的の位置にドロップ

### 4.2 要素の選択

- **単一選択:** 要素をクリック
- **複数選択:** `Shift + クリック` または `Ctrl + クリック`
- **選択解除:** キャンバスの空白部分をクリック、または `Escape` キー

### 4.3 要素の移動とリサイズ

- **移動:** 選択した要素をドラッグ
- **リサイズ:** 要素の周囲に表示される8つのハンドルをドラッグ
- **ロック:** ロックされた要素は移動・リサイズできません（レイヤーパネルで設定）

### 4.4 プロパティの編集

要素を選択すると、右サイドバーの **「プロパティ」** タブに設定が表示されます。

**共通プロパティ:**
- サイズ (幅・高さ)
- 位置 (X・Y)
- ボーダー (色・太さ・スタイル・角丸)
- 背景色
- 透明度
- 表示条件

### 4.5 要素の操作

| 操作 | ショートカット |
|------|--------------|
| コピー | `⌘C` / `Ctrl+C` |
| 切り取り | `⌘X` / `Ctrl+X` |
| 貼り付け | `⌘V` / `Ctrl+V` |
| 複製 | `⌘D` / `Ctrl+D` |
| 削除 | `Delete` または `Backspace` |
| Undo | `⌘Z` / `Ctrl+Z` |
| Redo | `⌘⇧Z` / `Ctrl+Shift+Z` |

右クリックでコンテキストメニューも表示されます。

## 5. 要素タイプ別ガイド

### 5.1 テキスト要素

テキストを自由に配置できます。動的データを差し込む場合は `{{フィールドキー}}` 形式を使います。

**例:**
```
注文No: {{order_id}}
お客様名: {{customer.name}} 様
```

**プロパティ:**
- フォント・サイズ・スタイル（太字・斜体・下線）
- 文字色・背景色
- 配置（左揃え・中央・右揃え）
- 縦書き
- ふりがな設定

**インライン編集:** 要素をダブルクリックするとテキストを直接編集できます。

### 5.2 データフィールド要素

単一のデータフィールドを表示します。フォーマット設定が豊富です。

**プロパティ:**
- フィールドキー（ドット記法で階層指定可）
- 表示フォーマット（数値・日付・通貨・漢数字等）
- fallback テキスト（データがない場合の表示）

**使用例:**
```
フィールドキー: order.amount
フォーマット: 通貨（¥）
→ ¥1,234,567 と表示
```

### 5.3 繰り返しバンド要素

配列データをテーブル形式で繰り返し表示します。

**設定手順:**
1. 繰り返しバンドをキャンバスに追加
2. プロパティパネルで「データソース」を配列フィールドキーに設定（例: `orders`）
3. 列を定義（フィールドキー・ラベル・幅・フォーマット）
4. ヘッダー・フッターの表示設定
5. 集計（合計・平均等）の設定

### 5.4 グラフ要素

棒グラフ・折れ線グラフ・円グラフなどを表示します。

**設定手順:**
1. グラフをキャンバスに追加
2. グラフの種類を選択（棒/折れ線/円/ドーナツ）
3. データソース（配列フィールドキー）を設定
4. X軸・Y軸フィールドを指定

### 5.5 バーコード要素

QRコードや各種バーコードを表示します。

**対応形式:**
- QR コード
- CODE 128
- CODE 39
- JAN 13 (EAN 13)

**プロパティ:**
- バーコード種類
- 値（静的テキストまたは `{{フィールドキー}}` で動的設定）
- バーコード下のテキスト表示

### 5.6 帳票テーブル要素

固定ヘッダーとデータ行が混在する複合表です。

**特徴:**
- ヘッダー行・データ行・フッター行を自由に組み合わせ
- セル結合
- 各セルに独立したデータバインディング

### 5.7 印鑑 (ハンコ) 要素

日本語帳票に使う印鑑を表示します。

**プロパティ:**
- テキスト（氏名など）
- 形状（丸・角・楕円）
- ボーダー色
- 書字方向（縦書き・横書き）

### 5.8 承認印欄要素

複数の承認印を横並びで表示します。

**プロパティ:**
- セル数と各セルのラベル
- 印鑑画像の設定
- ラベル位置（上・下）

### 5.9 テナント情報要素

組織の情報（会社名・住所・ロゴ等）を自動で差し込みます。

**種類:**
- 会社名
- 住所
- 電話番号
- 代表者名
- ロゴ
- カスタムフィールド

テナント情報は「データ設定」→「テナント情報」タブで設定します。

## 6. データバインディング

動的データを帳票に差し込む機能です。

### 6.1 スキーマの定義

1. 左サイドバーの **「スキーマ」** タブを開く
2. **「フィールドを追加」** でフィールドを定義
3. フィールドキー・ラベル・型を設定

**マスターグループ:** 帳票全体で1件のデータ（会社情報・注文ヘッダー等）

**詳細グループ:** 繰り返し行のデータ（注文明細等）

### 6.2 テストデータの入力

1. 左サイドバーの **「データ」** タブを開く
2. JSON 形式でテストデータを入力

```json
{
  "customer_name": "株式会社サンプル",
  "order_date": "2026-04-12",
  "items": [
    { "name": "商品A", "qty": 2, "price": 1000 },
    { "name": "商品B", "qty": 1, "price": 5000 }
  ]
}
```

3. キャンバスにリアルタイムでデータが反映されます

### 6.3 フィールドキーの書き方

| 書き方 | 説明 | 例 |
|--------|------|---|
| `フィールド名` | 単純なフィールド | `name` |
| `グループ.フィールド名` | 階層フィールド | `customer.name` |
| `{{フィールド名}}` | テキスト要素内でのトークン | `{{company_name}}` |

**システム変数:**
| 変数 | 説明 |
|------|------|
| `{{$page}}` | 現在のページ番号 |
| `{{$totalPages}}` | 全ページ数 |
| `{{$printDate}}` | 印刷日 |

### 6.4 ScalarDB との連携

1. スキーマパネルで **「ScalarDB カタログ」** を開く
2. 接続する名前空間・テーブルを選択
3. 各フィールドをテーブルのカラムにマッピング
4. サーバーからデータを取得してプレビューで確認

## 7. 計算ルール

データフィールドを計算する JEXL 式を定義できます。

### 7.1 計算ルールの追加

1. ツールバーの **「データ設定」** ボタンをクリック
2. **「計算フィールド」** タブを開く
3. **「ルールを追加」** をクリック
4. キー・ラベル・計算式を入力

### 7.2 計算式の例

| 式 | 説明 |
|----|------|
| `price * qty` | 掛け算 |
| `subtotal * 1.1` | 消費税込み計算 |
| `sum(items.price)` | 配列の合計 |
| `round(amount, 0)` | 四捨五入 |
| `ifExpr(qty > 0, "在庫あり", "在庫なし")` | 条件分岐 |
| `formatDate(date, "yyyy/MM/dd")` | 日付フォーマット |
| `concat(last_name, " ", first_name)` | 文字連結 |

**利用可能な関数:**

| 関数 | 説明 |
|------|------|
| `sum(array)` | 配列の合計 |
| `count(array)` | 配列の件数 |
| `avg(array)` | 配列の平均 |
| `min(array)` | 配列の最小値 |
| `max(array)` | 配列の最大値 |
| `round(n, decimals)` | 四捨五入 |
| `concat(...)` | 文字列結合 |
| `formatDate(date, pattern)` | 日付フォーマット |
| `formatNumber(n, pattern)` | 数値フォーマット |
| `ifExpr(cond, a, b)` | 条件式 |
| `Math.abs/ceil/floor/round/max/min/pow/sqrt` | 数学関数 |

## 8. バリデーションルール

入力値の検証ルールを定義できます。

1. 「データ設定」→ **「入力検証」** タブを開く
2. 違反条件を JEXL 式で記述
3. 違反時のメッセージと重要度を設定

**例:**
```
条件式: amount <= 0
メッセージ: 金額は0より大きい値を入力してください
重要度: エラー
```

## 9. プレビューと出力

### 9.1 プレビューモード

ツールバーの **「プレビュー」** ボタンをクリックすると、データが差し込まれた状態の帳票を確認できます。

プレビュー中は要素の編集はできません。もう一度クリックすると編集モードに戻ります。

### 9.2 ライブプレビュー (分割表示)

「プレビュー」ボタン右の **▼** をクリック → 「ライブプレビューを表示」で、エディタとプレビューを並べて表示できます。

### 9.3 PNG エクスポート

ツールバーの **「PNG」** ボタンをクリックすると、現在のページを PNG 画像として保存します。

### 9.4 PDF エクスポート

ツールバーの **「PDF」** ボタンをクリックすると、全ページを PDF として保存します。

**PDF エクスポート右の ▼ メニューから:**
- ページを選択してエクスポート
- サーバーサイド PDF 生成（より高品質）

### 9.5 JSON エクスポート

「保存」ボタン右の **▼ メニュー** から **「JSONとしてダウンロード」** で帳票定義ファイルをエクスポートできます。バックアップや他環境へのインポートに使用します。

## 10. 保存とバージョン管理

### 10.1 保存

- **クイック保存:** ツールバーの **「保存」** ボタン (`⌘S`)
- **名前を付けて保存:** 「保存」右の **▼** → 「名前を付けて保存」
- **ローカル保存:** 自動的に LocalStorage に保存されます（1秒ごと）

初回保存時はテンプレート名の入力を求められます。

### 10.2 バージョン履歴

右サイドバーの **「バージョン」** タブでバージョン履歴を管理できます。

- **バージョンを作成:** 重要な変更前にスナップショットを保存
- **バージョンを復元:** 過去のバージョンに戻す（元のバージョンは失われません）

### 10.3 自動保存の復元

アプリを開いた際に前回の作業が自動保存されていた場合、復元するか確認のダイアログが表示されます。

## 17. よくある質問

**Q: テンプレートを開く方法は？**  
A: ツールバーの「開く ▼」→「サーバーから開く」でサーバーに保存されたテンプレートを開けます。「ローカルファイルを開く」でローカルの JSON ファイルを読み込めます。

**Q: 保存したテンプレートが見つからない**  
A: ツールバーの「テンプレート管理」→「バックエンドテンプレート」→「一覧を取得」をクリックしてください。

**Q: データが帳票に反映されない**  
A: 左サイドバーの「データ」タブで正しい JSON 形式のテストデータが入力されているか確認してください。フィールドキーのスペルミスがないかも確認してください。

**Q: PDF 出力の品質が低い**  
A: サーバーサイド PDF 生成を使用してください。「PDF ▼」→「サーバーサイド PDF 生成」を選択します（バックエンドへの接続が必要）。

**Q: 印刷時に余白がずれる**  
A: ページ設定で正確な余白を設定し、ブラウザの印刷ダイアログで「余白なし」を選択してください。PDF エクスポートを使用するとより正確です。

**Q: 「ログイン試行回数が多すぎます」と表示される**  
A: 5分間のレートリミットがかかっています。しばらく待ってから再試行してください。

**Q: Undo でフォーミュラ変更が元に戻らない**  
A: 計算ルール・バリデーションルールの変更は Undo/Redo の対象外です。これは設計上の仕様です。

## 18. トラブルシューティング

### バックエンドに接続できない

画面右上に「バックエンド未接続」と表示されている場合:

1. バックエンドサーバーが起動しているか確認: `npm run dev:backend`
2. `http://localhost:8080/api/v1/auth/me` にアクセスして応答を確認
3. ファイアウォールやセキュリティソフトが8080番ポートをブロックしていないか確認

### テンプレートが保存できない

1. バックエンドへの接続を確認（上記参照）
2. `server/scalardb.properties` が正しく設定されているか確認
3. データベースファイルへの書き込み権限を確認

### ブラウザのコンソールにエラーが表示される

`F12` でデベロッパーツールを開き、「コンソール」タブのエラーメッセージを確認してください。エラーの内容と再現手順をシステム管理者に報告してください。
