## 1. はじめに

Report Design Studio V2 は、請求書・見積書・税務様式などの帳票やフォームをブラウザ上でビジュアルに設計し、PDF/PNG として出力できる Web アプリケーションです。

主な特徴:

- **ドラッグ&ドロップ**で要素を配置
- **データバインディング**で `{{fieldKey}}` トークンによる動的データ差し込み
- **ScalarDB 連携** — テーブルスキーマの取得・データ解決・データブラウザでの行編集
- **JEXL 式**による計算・集計と**バリデーションルール**
- **出力バリアント**で用途別 PDF（マスキング・要素の表示切替）を一括生成
- **サーバーサイド PDF** — ページ分割・和文フォント埋め込み・縦書き・ふりがな対応の高品質出力
- **日本語帳票専用要素** — 印鑑・承認欄・収入印紙・元号選択
- **バージョン履歴**で過去状態への復元
- **フォーム回答収集**と Excel/PDF エクスポート

### 動作環境

- **ブラウザ:** Google Chrome (最新版推奨)、Edge、Safari
- **URL:** `http://localhost:5173`（開発環境の既定）

## 2. ログイン

アプリを開くとログイン画面が表示されます。

1. **ユーザーID** に `admin` を入力
2. **パスワード** に `changeme`（初期値）を入力
3. **ログイン** ボタンをクリック

> **注意:** ログイン試行は5分間に5回までです。上限を超えるとしばらく待つ必要があります。

> **セキュリティ:** 初期パスワードのまま外部公開しないでください。初回ログイン後に管理タブからパスワードを変更するか、初回起動前に `ADMIN_PASSWORD` 環境変数を設定してください。UI から変更したパスワードは再起動で巻き戻りません。

ログインに成功すると、帳票デザイナー画面（デザインタブ）に移動します。

## 3. 画面構成

### 3.1 トップナビゲーション（6 タブ）

画面上部のナビゲーションで機能領域を切り替えます。タブは「編集」「リソース管理」「システム管理」の 3 グループに区切られています。

| タブ | 内容 |
|------|------|
| **デザイン** | 帳票デザイナー本体（キャンバス + サイドバー） |
| **バインド** | スキーマフィールドと要素のバインディングを可視化・編集する 3 ペインエディタ |
| **テンプレート管理** | サーバー保存テンプレートの一覧・複製・公開範囲・インポート/エクスポート |
| **回答** | フォーム回答の一覧・PDF/Excel エクスポート |
| **データブラウザ** | ScalarDB テーブルの閲覧・行の追加/編集/削除 |
| **管理** | ユーザー管理・サーバー設定（admin ロールが必要） |

### 3.2 デザインタブの構成

## 4. テンプレートの作成・読み込み

### 4.1 白紙から作成

1. ツールバー左端の **「新規作成」** ボタンをクリック
2. 白紙の帳票が表示されます（未保存の変更は事前に保存してください）

### 4.2 テンプレートから開始

1. トップナビの **「テンプレート管理」** タブ（またはツールバーのテンプレートボタン）を開く
2. ビルトインテンプレートやサーバー保存テンプレートの一覧から選択して読み込みます

**利用可能なビルトインテンプレートの例:**
- 見積書（インボイス対応）
- 見積書（値引対応・インボイス対応）
- Quotation (English)
- 扶養控除等申告書（令和7年分）
- 要素ショーケース（全要素の確認用）

### 4.3 サーバー/ファイルから開く

- ツールバーの **「開く ▼」→「サーバーから開く」** — サーバー保存テンプレートを開く
- **「開く ▼」→「ローカルファイルを開く」** — `.json`（エンベロープ形式）ファイルを読み込む。旧形式のファイルも自動でマイグレーションされます

### 4.4 JSON としてエクスポート

「保存」ボタン右の **▼ メニュー** から **「JSONとしてダウンロード」** で帳票定義ファイル（`{ formatVersion: 2, definition }` エンベロープ形式）をエクスポートできます。バックアップや他環境へのインポートに使用します。

## 5. 要素の追加と配置

### 5.1 パレットからの追加

1. 左サイドバーの **「要素」** タブを開く
2. 追加したい要素のボタンをクリック → ページ中央付近に配置されます
3. またはボタンを **キャンバスへドラッグ&ドロップ**

### 5.2 要素の選択・移動・リサイズ

- **単一選択:** 要素をクリック
- **複数選択:** `Shift + クリック` または `Ctrl/⌘ + クリック`、キャンバス空白部分のドラッグで範囲選択
- **選択解除:** 空白部分をクリック、または `Escape`
- **移動:** ドラッグ、または矢印キーで 1mm 単位（`Shift+矢印` で 5mm）
- **リサイズ:** 要素の周囲に表示される 8 つのハンドルをドラッグ
- **ロック:** ロックされた要素は移動・リサイズ不可（レイヤーパネルで設定）

### 5.3 コピー・複製・削除

| 操作 | ショートカット |
|------|--------------|
| コピー | `⌘C` / `Ctrl+C` |
| 切り取り | `⌘X` / `Ctrl+X` |
| 貼り付け | `⌘V` / `Ctrl+V` |
| 複製 | `⌘D` / `Ctrl+D` |
| 削除 | `Delete` / `Backspace` |

右クリックでコンテキストメニューも表示されます。

## 6. 要素の種類と設定

全 24 種類の要素を配置できます。

### テキスト系

| 要素 | 説明 |
|------|------|
| **テキスト** | 固定文字列。`{{fieldKey}}` トークンでデータ埋め込み可。縦書き・ふりがな対応。ダブルクリックでインライン編集 |
| **データフィールド** | 単一フィールドを解決してフォーマット表示（数値・日付・通貨・漢数字等） |

**テキストへのデータ埋め込み例:**
```
請求先: {{customer.name}} 様
請求金額: {{total}} 円
```

**システム変数:**
| 変数 | 説明 |
|------|------|
| `{{$page}}` | 現在のページ番号 |
| `{{$totalPages}}` | 全ページ数 |
| `{{$printDate}}` | 印刷日 |

システム変数と和暦フォーマットはサーバー PDF でもサーバー側で解決されます。

### データ表示・繰り返し

| 要素 | 説明 |
|------|------|
| **グラフ** | 棒・折れ線・円・ドーナツ等（Recharts） |
| **繰り返しバンド** | 配列データを行として繰り返す帳票バンド（請求明細等）。サーバー PDF では枠を超えた行が自動で次ページにフローします |
| **繰り返しリスト** | 配列データをカード・グリッド形式で表示（商品カタログ等） |
| **帳票テーブル** | 固定ヘッダー + データ行の複合表。セル結合・Excel 風編集対応（→ [14 章](#14-帳票テーブルの編集)） |

### 図形・画像

| 要素 | 説明 |
|------|------|
| **図形** | 矩形・円・線。塗り・線色・角丸を設定可能 |
| **画像** | PNG/JPEG/WebP/GIF（Base64 または HTTPS URL）。データバインドで動的画像も可能 |
| **バーコード** | QR / CODE128 / CODE39 / JAN13 (EAN13)。値は静的テキストまたは `{{フィールドキー}}` |

### 帳票共通

| 要素 | 説明 |
|------|------|
| **ページ番号** | 自動ページ番号（カスタムフォーマット対応） |
| **日付** | 現在日付（和暦含むカスタムフォーマット対応） |
| **区切り線** | 水平/垂直、実線/破線/点線 |

### 記入欄

| 要素 | 説明 |
|------|------|
| **記入欄** | 手書き記入用のフィールド枠（グリッド/ライン表示） |
| **チェックボックス** | データバインディング対応のチェックボックス |

### 日本語帳票専用

| 要素 | 説明 |
|------|------|
| **印鑑（ハンコ）** | 社印・個人印。形状（丸/角/楕円）・縦書き/横書き |
| **承認印欄** | 承認フロー用の複数段印鑑ボックス。セル数・ラベル・印鑑画像を設定 |
| **収入印紙欄** | 収入印紙の貼付欄 |
| **元号選択** | 令和/平成/昭和/大正の選択欄 |

### テナント情報

組織の情報を自動で差し込む要素群です（→ [22 章](#22-テナント情報の設定)）。

- 会社名 / 住所 / 電話番号 / 代表者名 / ロゴ / カスタムフィールド

> **廃止された要素:** 旧「ラベル」は「テキスト」に、旧「テーブル」は「帳票テーブル」に統合されました。旧形式のテンプレートを読み込むと自動変換されます。

## 8. データスキーマとバインディング

### 8.1 スキーマの定義

1. 左サイドバーの **「スキーマ」** タブを開く
2. グループを作成
   - **マスター** — 帳票全体で 1 件のデータ（顧客情報・注文ヘッダー等）
   - **詳細** — 配列データ（明細行等）
3. グループ内にフィールドを定義
   - キー（英数字・アンダースコア）/ ラベル / 型（文字列・数値・日付・真偽値・配列・画像）

### 8.2 フィールドキーの書き方

| 書き方 | 説明 | 例 |
|--------|------|---|
| `フィールド名` | 単純なフィールド | `name` |
| `グループ.フィールド名` | 階層フィールド（ドット記法） | `customer.name` |
| `{{フィールド名}}` | テキスト要素内でのトークン | `{{company_name}}` |

### 8.3 テストデータの入力

ツールバーの **「データ設定」→「テンプレートデータ」** タブで JSON 形式のテストデータを入力します。キャンバスにリアルタイムで反映されます。

```json
{
  "customer_name": "株式会社サンプル",
  "order_date": "2026-07-17",
  "items": [
    { "name": "商品A", "qty": 2, "price": 1000 },
    { "name": "商品B", "qty": 1, "price": 5000 }
  ]
}
```

### 8.4 バインドタブ

トップナビの **「バインド」** タブでは、テンプレート要素（左）とスキーマフィールド（中央）の対応が色分けされた矢印で可視化されます。クリックまたはドラッグでバインドを作成・変更でき、右ペインで DB 接続を設定できます。

### 8.5 ScalarDB との連携

1. スキーマ定義でグループを ScalarDB の名前空間・テーブルに割り当てる
2. 各フィールドをテーブルのカラムにマッピングする
3. サーバーからデータを取得してプレビューで確認する

テーブルの中身は **データブラウザ** タブで直接確認・編集できます（→ [18 章](#18-データブラウザ)）。

## 9. 計算ルール

ツールバーの **「データ設定」→「計算フィールド」** タブで計算ルールを管理します。

### 9.1 計算ルールの追加

1. **「ルールを追加」** をクリック
2. 以下を設定:

| フィールド | 説明 | 例 |
|-----------|------|----|
| **キー** | 結果を参照するためのキー（英数字・アンダースコア） | `total_price` |
| **ラベル** | 表示名 | 合計金額 |
| **式** | JEXL 式 | `sum(items.price)` |
| **結果型** | 数値 / 文字列 / 真偽値 | 数値 |
| **エラー時** | 0 を返す / 空にする / エラーテキスト | 0 を返す |

### 9.2 計算式の例

```
price * quantity                     // 乗算
subtotal * 1.1                       // 消費税込み
amount > 0 ? "請求あり" : "請求なし"  // 三項演算子
ifExpr(qty > 0, "在庫あり", "在庫なし") // 条件分岐
concat(last_name, " ", first_name)   // 文字連結
```

### 9.3 利用可能な関数

| 関数 | 説明 |
|------|------|
| `sum(array)` / `count(array)` / `avg(array)` / `min(array)` / `max(array)` | 配列集計 |
| `round(n, decimals)` | 四捨五入 |
| `concat(...)` | 文字列結合 |
| `formatDate(date, pattern)` / `formatNumber(n, pattern)` | フォーマット |
| `ifExpr(cond, a, b)` | 条件式 |
| `Math.abs/ceil/floor/round/max/min/pow/sqrt` | 数学関数 |

金額計算はサーバー側で BigDecimal を用いて評価されるため、浮動小数点誤差による 1 円ズレは発生しません。

### 9.4 テスト実行・変数参照

- 各ルールの **「テスト」** ボタンで現在のテストデータで式を評価し、結果またはエラーを表示します
- **「変数」** ボタンで参照可能な変数（スキーマのフィールド・他ルールのキー・組み込み関数）の一覧を表示し、クリックで式に挿入できます

### 9.5 セキュリティガード

以下の式はブロックされます:
- ホワイトリスト（上記関数）以外のクラス・メソッドへのアクセス
- `constructor`、`__proto__`、`prototype` を含む式
- 500 文字を超える式、深いネスト（16 段超）
- 評価が 500ms を超える式（タイムアウト）

### 9.6 キー重複の警告

同じキーを持つルールが 2 件以上あると警告が表示されます。キーはユニークにしてください。

## 10. バリデーションルール

ツールバーの **「データ設定」→「入力検証」** タブで管理します。

1. 違反条件を JEXL 式で記述
2. 違反時のメッセージと重大度（error / warning）を設定

**例:**
```
条件式: amount <= 0
メッセージ: 金額は0より大きい値を入力してください
重大度: エラー
```

### バリデーションの実行

ツールバーの **「バリデーション実行」** ボタンをクリックします。

- 全ルールがパスすると緑のバッジ
- 警告・エラーがあるとオレンジ・赤のバッジ + 件数が表示されます

## 11. 条件付き表示

各要素は特定条件を満たす場合のみ表示/非表示にできます。

1. 対象要素を選択
2. 右サイドバーの **「条件付き表示」** セクションを開く
3. **「+ 条件追加」** で条件を追加:
   - フィールドを選択
   - 演算子（`==`, `!=`, `>`, `<`, `>=`, `<=`, `contains` など）を選択
   - 比較値を入力
4. 複数条件がある場合、**AND / OR** の結合方法を選択

**使用例:** 「消費税額」テキストを `tax_included == true` のときのみ表示。

## 14. 帳票テーブルの編集

帳票テーブル（FormTable）要素はキャンバス上で Excel 風の直接編集ができます。

| 操作 | 方法 |
|------|------|
| **編集モードに入る** | テーブルをダブルクリック（`Esc` または外側クリックで解除） |
| **セル選択** | クリック / `Shift+クリック` / 矢印キー / `Tab` で選択・移動 |
| **セルの編集** | セルをダブルクリックまたは `Enter` — タイプ・テキスト・スタイルをポップオーバーで編集 |
| **行列の挿入/削除/移動** | 右クリックメニュー |
| **セル結合** | 複数セルを選択して右クリック → 結合（解除も同様） |
| **列幅・行高の変更** | 列/行の境界をドラッグ（最小 3mm） |
| **コピー & ペースト** | `Ctrl+C/V/X` — テーブル内部および Excel との TSV 相互コピペに対応 |
| **Undo** | 編集モード中はテーブル専用の Undo。モードを抜けると 1 回の操作として全体の履歴に統合されます |

各セルには独立したデータバインディングを設定でき、`dataSource` に配列フィールドを指定するとデータ行が繰り返されます。

## 16. プレビューと出力

### 16.1 プレビューモード

ツールバーの **「プレビュー」** ボタンでプレビューモードに切り替えます。データバインドが適用された状態でレンダリングされ、要素の編集はできません。もう一度クリックで編集モードに戻ります。

プレビューメニューの **「フルプレビュー（PDF）」** で、サーバーで生成した実際の PDF をそのまま確認できます（ページ分割の結果も確認可能）。

### 16.4 エクスポート

ツールバーの **「エクスポート」** ドロップダウンから選択します。

| メニュー | 内容 |
|---------|------|
| **PNG** | 現在のページを PNG 画像として保存 |
| **PDF（全ページ）** | クライアントサイドで全ページを PDF 化（バックエンド不要のフォールバック） |

### 16.5 ページ分割とあふれ警告

- 繰り返しバンド/リストのデータが要素枠に収まらない場合、デザイナー上に **あふれ警告バッジ** が表示されます
- **サーバー PDF ではあふれた行は自動的に次ページへフロー**します（ヘッダーは継続ページにも繰り返し描画）。明細テーブルの繰越小計（「次頁へ続く」「前頁より繰越」）やグループ改ページも設定できます
- クライアントサイド PDF（フォールバック）はページ分割を行わず、枠内でクリップされます

## 25. よくある質問

**Q: テンプレートを開く方法は？**  
A: ツールバーの「開く ▼」→「サーバーから開く」でサーバーに保存されたテンプレートを開けます。「ローカルファイルを開く」でローカルの JSON ファイルを読み込めます。トップナビの「テンプレート管理」タブからも一覧・読み込みができます。

**Q: データが帳票に反映されない**  
A: 「データ設定」→「テンプレートデータ」で正しい JSON 形式のテストデータが入力されているか、`{{fieldKey}}` のスペルがスキーマのキーと一致しているかを確認してください。

**Q: PDF 出力の品質が低い / 文字がぼやける**  
A: **「エクスポート」→「PDF（サーバー生成）」** を使用してください（バックエンドへの接続が必要）。クライアントサイド PDF は画像ベースのフォールバックです。

**Q: 明細行が多くて 1 ページに収まらない**  
A: サーバー PDF はあふれた行を自動で次ページに分割します。デザイナー上のあふれ警告バッジは「サーバー PDF 出力ではページ分割される」ことを示しています。

**Q: 印刷時に余白がずれる**  
A: ページ設定で正確な余白を設定し、ブラウザの印刷ダイアログで「余白なし」を選択してください。PDF エクスポートを使用するとより正確です。

**Q: 「ログイン試行回数が多すぎます」と表示される**  
A: 5 分間のレートリミットがかかっています。しばらく待ってから再試行してください。

**Q: Undo で計算式の変更が元に戻らない**  
A: 計算ルール・バリデーションルールの変更は Undo/Redo の対象外です（設計上の仕様）。

**Q: 古い JSON ファイルは読み込める？**  
A: はい。旧形式（`$schema: "report-definition/v1"` や旧 `Report` 形式）は読み込み時に自動でマイグレーションされます。より新しい `formatVersion` のファイルは読み込めません。

## 26. トラブルシューティング

### 要素が動かない

- **プレビューモード** になっていないか確認
- ロックされていないかレイヤーパネルで確認
- グリッドスナップが有効の場合、意図しない位置にスナップすることがあります

### バックエンドに接続できない

ステータスバーに「切断」と表示されている場合:

1. バックエンドサーバーが起動しているか確認: `npm run dev:backend`
2. `http://localhost:8080/api/v1/health` にアクセスして応答を確認
3. ファイアウォールやセキュリティソフトが 8080 番ポートをブロックしていないか確認

### テンプレートが保存できない

1. バックエンドへの接続を確認（上記参照）
2. `server/scalardb.properties` が正しく設定されているか確認
3. データベースファイルへの書き込み権限を確認

### 計算式がエラーになる

- 「テスト」ボタンで式を評価してエラー内容を確認
- キーが英数字・アンダースコアのみであるか確認（日本語キー不可）
- 使用できる関数は [9.3 節](#93-利用可能な関数) の一覧のみです
- 式が 500 文字を超えていないか確認

### PDF 書き出しに失敗する

- サーバー生成 PDF の場合、バックエンドが起動しているか確認
- バックエンドログでエラーを確認
- ページ数・要素数が多い場合は非同期ジョブ（時間がかかる場合があります）

### ブラウザのコンソールにエラーが表示される

`F12` でデベロッパーツールを開き、「コンソール」タブのエラーメッセージを確認してください。エラーの内容と再現手順をシステム管理者に報告してください。
